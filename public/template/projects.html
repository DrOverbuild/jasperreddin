School Projects

ChessGame

Built as a team project for CSCI4490, the UCA Computer Science Department capstone course. It is a nearly complete implementation of the game of chess allowing two separate computers to connect to a server, enabling the two players to play against each other on separate computers. It makes use of the Object Server-Client Framework, a TCP framework that allows socket communication of Java serializable objects. The server connects to a MySQL database to keep track of players' login information and statistics.

I was the team leader for this project, brainstorming ideas and coordinating work among three other contributors. My main contribution was the game logic. The implementation of chess is not an easy thing to do, with there being several different moves across thousands of conditions. My design was strongly object-oriented, using polymorphism to build an initial list of moves each piece can make regardless of the other pieces on the board. After that list was generated, we eliminated possible moves by checking if the piece was blocked by other pieces. Finally, we removed moves on the board that put the king in check. This last step required us to copy the board in memory, make the hypothetical move, and check if the King is endangered. After that final step, if there were not any moves left for the player, we considered that the end of the game.

The multiplayer communication was built using OCSF, a framework that provides all of the basic TCP/IP socket communication.

TicTacToe

This was a project for CSCI 4300, the Operating Systems course at UCA. The final project focused on socket inter-process communication. We were required to implement a multiplayer game using whichever programming language we prefered. So of course I chose Java because I had already gotten a chance previously to play with sockets and streams in Java. One may be thinking, well, anyone can do a TicTacToe game, there's nothing special about that, but the beauty of this project lies in the design of the TCP/IP socket communication between the clients and the server. The GitHub project includes a full report and documentation of the project.
